# FEED ENGINE SIMPLIFICATION AGENT

## Objective
Simplify AquaRythu feed engine to single deterministic flow for V1 launch.

## Scope

### ✅ ALLOWED MODIFICATIONS
- `lib/core/engines/feed/*` (all files)
- `lib/core/services/feed_service.dart` (if removing old calls only)

### ❌ FORBIDDEN
- UI files (`lib/features/*`, `lib/widgets/*`)
- Providers (`lib/core/providers/*`)
- Dashboard/screens
- Supabase logic
- Routes configuration

## Core Responsibilities

### 1. Replace Multi-Engine Pipeline
Convert current architecture into single flow:
```
Base Feed (existing compute logic)
  ↓
Tray Factor (simple & deterministic)
  ↓
Apply Factor
  ↓
Safety Clamp (±30% from lastFeed)
  ↓
Return OrchestratorResult
```

### 2. Disable Old Engines (Systematically)
- SmartFeedEngineV2
- FeedIntelligenceEngine
- FeedDecisionEngine
- FeedRecommendationEngine

Comment out calls, add `// ❌ DISABLED FOR V1` markers.

### 3. Enforce Single Entry Point
- MasterFeedEngine must be only feed logic entry
- Remove all alternate paths
- FeedOrchestrator.dart calls only MasterFeedEngine

## Error Handling Rules

### Unsafe Patterns (REPLACE ONLY THESE)
- `throw Exception(...)` (feed context)
- `return 0;` (when feed value expected)

### Safe Replacement
Replace with:
```dart
trayFactor = 1.0;
// Continue execution
```

### MANDATORY Logging
For every replacement:
1. Log before/after code
2. Show reason for change
3. Do NOT silently override logic

## Testing Rules

### Test Cases (MANDATORY)
Implement validation for:
1. **DOC 10** — Blind stage, feed increases smoothly
2. **DOC 35 no tray** — Feed > 0, trayFactor = 1.0
3. **Full tray** — Feed decreases ~15%
4. **Empty tray** — Feed increases ~10%
5. **Spike clamp** — lastFeed=10kg, calc=25kg → final≤13kg

### Verification Required
- [ ] No crash
- [ ] Feed never = 0 unexpectedly
- [ ] Outputs are deterministic (same input → same output)
- [ ] All test cases pass

## Completion Criteria (GATE)

Agent MUST verify before marking done:

✅ All feed flows use MasterFeedEngine only  
✅ No SmartFeedEngineV2 execution paths  
✅ No intelligence/decision/recommendation layers  
✅ Zero feed spike bug eliminated  
✅ Test cases pass (5/5)  
✅ Deterministic output confirmed  
✅ No scope violations (UI/providers untouched)  

## Constraints

- **Do not** over-engineer or add new features
- **Do not** redesign architecture
- **Do not** touch unrelated systems
- **Do not** modify tests in `test/legacy/*`
- **Focus only** on simplification + validation
